Key Takeaways

What I Learned
from This Project

01
Modular & Scalable Code

Learned to structure ML projects into separate components — data ingestion, transformation, and model training — each in its own file with clean interfaces. This makes the codebase easy to debug, test, and extend without breaking other parts.

OOP Pipelines Exception Handling Logging
02
End-to-End ML Pipeline

Built a full pipeline from raw CSV to live predictions — including preprocessing with ColumnTransformer, model selection with GridSearchCV across 7 algorithms, serialisation with pickle, and serving via Flask.

scikit-learn GridSearchCV ColumnTransformer Flask
03
AWS Elastic Beanstalk Deployment

Deployed a Python Flask app to AWS Elastic Beanstalk, learning how EB handles environment setup, gunicorn configuration, WSGI path settings, Procfile parsing, and platform-specific quirks like Amazon Linux 2023.

Elastic Beanstalk Gunicorn WSGI .ebextensions
04
IAM Permissions & AWS Security

Learned how IAM roles and policies control what AWS services can do on your behalf. Debugged missing permissions like elasticbeanstalk:CreateApplicationVersion and understood the principle of least privilege in practice.

IAM Policies Service Roles Permissions
05
CI/CD with AWS CodePipeline

Set up a continuous deployment pipeline that automatically deploys every GitHub push to production. Learned about pipeline stages, automated rollbacks, execution modes, and how to debug failed deployments from logs.

CodePipeline GitHub Integration CI/CD Auto Deploy
06
MLOps Fundamentals

Got hands-on experience with MLOps concepts — treating ML models as software artifacts, versioning them, automating deployment, monitoring via logs, and understanding the gap between a notebook model and a production system.

MLOps Model Versioning Production ML Log Debugging
